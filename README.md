## 1. Описание продукта
Тестируемая страница — конструктор виджета **«Календарь мероприятий»** от 3S.INFO (iGaming-портал). Виджет позволяет настроить параметры (тематика, страны, размер блока, цветовая тема) и получить готовый iframe-код для встраивания на сторонний сайт. Доступен на двух языках: русском (`/eventswidget/`) и английском (`/en/eventswidget/`).

## 2. Описание проверенных сценариев
* **2.1. Открытие страницы на русском языке:** Страница загружается, конструктор отображается корректно.
* **2.2. Переключение языка на английский:** Проверка отображения `/en/eventswidget/`.
* **2.3. Шаг 1 — выбор тематики (дропдаун):** Одиночный/множественный выбор, «Выбрать все», «Очистить все». Параметр `event_type` в коде.
* **2.4. Шаг 2 — выбор стран:** Одиночный и множественный выбор. Параметр `country` в коде.
* **2.5. Шаг 3 — ввод размеров (ширина/высота):** Ручной ввод, слайдер, граничные значения (0, отриц., макс.), буквы, спецсимволы, пустое поле.
* **2.6. Шаг 3 — чекбоксы «на всю ширину/высоту»:** Каждый по отдельности и оба одновременно. Значения `width`/`height` в коде.
* **2.7. Шаг 4 — все цветовые темы:** Синяя, зелёная, красная (светлые); бирюзовая, фиолетовая (тёмные). Параметр `theme=` в коде.
* **2.8. Генерация без выбора тематики:** Нажатие «Сгенерировать превью» при пустом Шаге 1.
* **2.9. Консоль — русская версия:** Анализ ошибок и предупреждений в DevTools Console.
* **2.10. Консоль — английская версия:** Анализ ошибок при открытии `/en/eventswidget/`.
* **2.11. Network — анализ ресурсов:** Статусы загрузки файлов, выявление 404 и Mixed Content.
* **2.12. Адаптивность (iPhone / iPad):** Проверка через Responsive Design Mode Safari.
* **2.13. Переход по событию из превью:** Открытие страницы события. Блок «Другие материалы».
* **2.14. Карусель промокодов в футере:** Проверка отображения карточек.

## 4. Баг-репорты

### 🟥 BUG-001: На английской версии вверху страницы отображается пустой блок изображения/анимации `Major`
* **URL:** `https://3s.info`
* **Шаги воспроизведения:**
  1. Открыть `https://3s.info`
  2. Переключить язык на English
  3. Открыть DevTools → Network
* **Ожидаемый результат:** Страница отображается корректно на английском языке
* **Фактический результат:** Большой пустой блок в верхней части страницы. При скролле вниз — вся страница функционирует корректно (конструктор работает). 
* **Причина:** GIF-файлы (`/uploads/media/c1/df/d9/6.gif` и `/uploads/media/90/2b/a3/7.gif`) возвращают 404 — изображение-заставка не загружается (не ломает функциональность, но портит первое впечатление).

---

### 🟥 BUG-002: Mixed Content: изображения промокодов по HTTP на HTTPS-странице — карточки пустые `Major`
* **URL:** `https://3s.info` (обе языковые версии)
* **Шаги воспроизведения:**
  1. Открыть `https://3s.info`
  2. Прокрутить до секции промокодов в футере
  3. Открыть DevTools → Console
* **Ожидаемый результат:** Все карточки промокодов отображают изображения
* **Фактический результат:** 3 из 4 карточек пустые (чёрные блоки). В консоли 10+ предупреждений Mixed Content.
* **Источник:** `footer-promocodes-slider.js` — ссылки на изображения прописаны по HTTP. Баг воспроизводится на обеих языковых версиях (RU и EN).

---

### 🟥 BUG-003: Тема «Красная»: параметр theme отсутствует в сгенерированном коде iframe `Critical`
* **URL:** `https://3s.info`
* **Шаги воспроизведения:**
  1. Открыть `https://3s.info`
  2. Шаг 4 → выбрать красную тему (светлая, третий вариант)
  3. Посмотреть код в поле внизу страницы
* **Ожидаемый результат:** `src="...?theme=red&event_lang=ru"`
* **Фактический результат:** `src="...?event_lang=ru"` — параметр theme полностью отсутствует. Все остальные темы передают `theme=` корректно.
* **Влияние:** пользователь получит дефолтную тему вместо красной.

---

### 🟨 BUG-004: Лейбл «Предыдущий пост» визуально наезжает на карточку в блоке «Другие материалы» `Minor`
* **URL:** Страница отдельного события на `dev.3s.info`
* **Шаги воспроизведения:**
  1. Из превью виджета перейти на любое событие
  2. Прокрутить до блока «Другие материалы» внизу страницы
* **Ожидаемый результат:** Лейбл и карточка отображаются без наложения
* **Фактический результат:** Текст «Предыдущий пост» и дата визуально перекрывают карточку. CSS-баг (overflow/z-index).

---

### 🟨 BUG-005: Import map конфликты JS-модулей в консоли (русская версия) `Minor`
* **URL:** `https://3s.info`
* **Шаги воспроизведения:**
  1. Открыть `https://3s.info`
  2. DevTools → Console → фильтр Предупреждения
* **Ожидаемый результат:** Нет конфликтов модулей в консоли
* **Фактический результат:** 10+ предупреждений: `"An import map rule for 'app' was removed, as it conflicted..."`
* **Затронуты:** `app`, `@symfony/stimulus-bundle`, `csrf_protection_controller.js` и др. Функциональность не нарушена, но указывает на проблему в конфигурации JS-сборки.
